# CodeLeap前端开发规范

## 1. 文件命名规范
- 文件夹：全部小写，多个单词用短横线连接（例如：login、task-list）
- Vue组件：大驼峰命名（例如：Login.vue、TaskList.vue）
- JS文件：全部小写，多个单词用短横线连接（例如：request.js、router.js）

## 2. 代码格式规范
- 使用2个空格缩进
- 语句末尾必须加分号
- 字符串使用单引号
- 大括号前后必须加空格

## 3. 组件命名规范
- 自定义组件使用大驼峰命名
- 组件名应该是多个单词，避免与HTML原生标签冲突

## 4. 注释规范
- 每个组件顶部必须添加组件说明注释
- 每个方法必须添加功能说明注释
- 复杂逻辑必须添加行内注释

## 5. 样式规范
- 优先使用class选择器，避免使用id选择器
- 全局样式放在`src/styles/global.css`中
- 组件样式必须添加`scoped`属性，避免样式污染
